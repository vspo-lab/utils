# Overview

A skill that reads specification documents generated by `/plan-feature` and produces a phased implementation checklist (`CHECKLIST.md`).

# Steps

## Step 1: Read Specs

1. Confirm the target feature name (`docs/plan/<feature>/` directory name) with the user
2. Read all `.md` files in `docs/plan/<feature>/`
3. Verify spec consistency (no undefined references or contradictions)

## Step 2: Generate Checklist

Generate `docs/plan/<feature>/CHECKLIST.md` following the checklist structure template in `docs/plan/README.md`.

- Populate each phase's checklist based on the spec content
- Use the 2-phase structure: Phase 1 (Implementation) and Phase 2 (Integration)
- If the spec contains `TBD` items, annotate the corresponding tasks with `[TBD]`

## Step 3: Present Implementation Guide

After generation, present the following:

1. Overview of all phases and estimated task count
2. The first phase to start (Phase 1: Implementation)
3. Dependencies between phases
4. Reminder: if a spec change is needed, update the spec document first, then modify code

# Rules

- Implementation order: type definitions -> core logic -> tests -> build verification
- Include test commands in each phase
- Generate Session Notes in an empty state (to be filled during implementation)
- Session Notes must always contain Done / Next / Risks & TODO
